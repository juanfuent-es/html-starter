1. Punto de partida

La web es principalmente texto. Aunque existan video, animación, inteligencia artificial y entornos inmersivos, la interacción digital sigue mediada por lenguaje escrito.

Texto no es decoración. Texto es interfaz. Diseñar web es organizar información escrita para que pueda leerse, escanearse y entenderse rápidamente.

2. Tipografía como sistema

Elegir una tipografía no es hacer tipografía. La tipografía en web implica construir un sistema.

  • Jerarquía
  • Escala
  • Ritmo vertical
  • Longitud de línea
  • Espaciado
  • Contraste
  • Accesibilidad

La calidad del diseño depende del sistema, no de la fuente aislada.

3. Familias tipográficas

Serif

Tradición editorial, adecuada para lectura extensa. Asociada a libros y publicaciones culturales. 

font-family: "Merriweather", serif;

Sans Serif

Contemporánea, funcional y altamente legible en pantalla. Dominante en interfaces digitales. 

font-family: "Inter", sans-serif;

Monospace

Caracteres de ancho igual. Asociada a programación y sistemas técnicos. 

font-family: "Fira Code", monospace;

Otros grupos genéricos

  • cursive: Simula escritura manual. Uso decorativo.
  • fantasy: Decorativa o temática. Uso expresivo puntual.
  • system-ui: Fuente nativa del sistema. Alta legibilidad y rendimiento.

4. Jerarquía y semántica

HTML define estructura semántica:

  • h1 → Nivel principal
  • h2 → Segundo nivel
  • h3 → Tercer nivel
  • p → Texto base

La jerarquía no depende solo del tamaño visual, sino de la estructura y el contraste.

5. Tamaño base y rem

El navegador define un tamaño base aproximado de 16px, que puede cambiar según la preferencia del usuario. 

html {
  font-size: 100%;
}

1rem equivale al tamaño raíz. Usar rem permite que el sistema escale proporcionalmente.

6. Unidades en tipografía

Absolutas

px — Unidad fija.

Relativas

  • rem — Relativa al elemento raíz.
  • em — Relativa al elemento padre.
  • % — Relativa al contenedor.
  • vw — Relativa al viewport.
  • ch — Relativa al ancho del carácter "0".

7. Longitud de línea

La legibilidad depende de la longitud de línea. Demasiado larga genera fatiga; demasiado corta fragmenta el ritmo.

Rango recomendado: 45–75 caracteres por línea.

article {
  max-width: 65ch;
}

8. Interlineado

El espacio entre líneas afecta la comodidad de lectura.

line-height: 1.5–1.7;

9. Evitar justificado en web

El texto justificado no es recomendable en web porque el ancho es fluido y puede generar espacios irregulares. 

text-align: left;

Pantalla no es impresión.

10. Texto como interfaz

Tratar texto como interfaz implica claridad inmediata, escaneo rápido y sistema coherente.

Optimizar tipografía es optimizar accesibilidad, usabilidad y comprensión.

11. Accesibilidad tipográfica

  • Escalabilidad con rem
  • Respeto a preferencias del usuario
  • Alto contraste
  • Tamaño base mínimo de 1rem

12. Principios del curso

  1. Diseñar sistemas, no tamaños sueltos.
  2. Respetar la preferencia del usuario.
  3. Usar rem para jerarquía.
  4. Controlar longitud con ch.
  5. No justificar texto.
  6. Priorizar lectura sobre ornamentación.
  7. Recordar que texto es interfaz.
