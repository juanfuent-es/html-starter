Introducción

El propósito de este ejercicio es diseñar una landing page personal tipo "link hub", donde concentres tus enlaces sociales o profesionales en una sola interfaz.

Este proyecto no es sólo una lista de enlaces. Es un ejercicio de jerarquía visual, estructura semántica, layout y composición tipográfica, con énfasis en la construcción de una sección Hero como bloque principal.

Requisitos mínimos

  1. Al menos tres enlaces a redes sociales o recursos externos.
  2. Un enlace de contacto vía correo electrónico.
  3. Un enlace de contacto vía WhatsApp o teléfono.
  4. Una imagen de perfil, logo o avatar.
  5. Una sección Hero correctamente estructurada.

Deadline: 24 de Marzo de 2026

Sección Hero

El proyecto debe iniciar con una sección Hero que funcione como primer impacto visual, contenedor de identidad y punto de entrada a la navegación.

Debe incluir

  • Nombre o marca personal.
  • Descriptor o tagline.
  • Imagen (avatar, retrato o recurso gráfico).
  • Al menos un llamado a la acción (CTA).
  • Uso consciente de clases, ids y selectores CSS.
  • Aplicación de flex o grid para layout.

Se evaluará

  • Jerarquía visual clara.
  • Contraste adecuado.
  • Adaptabilidad en móvil.
  • Coherencia tipográfica.

Estructura y Semántica

  • Uso correcto de header, nav, main, section, article y footer.
  • Un solo h1 en todo el documento.
  • Jerarquía lógica de encabezados.
  • Uso correcto de atributos alt, aria-label y title.
  • Organización clara de carpetas y archivos.

Layout y Modelo de Caja

  • Comprensión de margin, padding y border.
  • Uso de display flex o grid.
  • Control de ancho máximo con max-width.
  • Ritmo vertical consistente.
  • No depender de br para generar espacios.

Mobile First

  • Diseñar primero para pantalla móvil.
  • Uso progresivo de media queries.
  • Áreas táctiles adecuadas (mínimo recomendado 44px).
  • Correcto comportamiento desde 320px de ancho.

Accesibilidad y Diseño

  • Contraste mínimo AA.
  • Estados visibles en hover, focus y active.
  • No depender sólo del color para comunicar estados.
  • Uso de rel="noopener" en enlaces externos.
  • Texto alternativo descriptivo en imágenes.

Multimedia

  • Imágenes optimizadas para web.
  • Uso de formatos ideales, png para transparencias / JPG sin transparencia. *Nuevos formatos, webp, avif.
  • Dimensiones controladas desde CSS.
  • Compresión adecuada para mejorar tiempos de carga.

Sistema Visual

  • Escala tipográfica coherente (h1, h2, texto base).
  • Uso consciente de peso, tamaño y contraste.
  • Consistencia en espaciados.
  • Estados interactivos claros en enlaces y botones.
  • Uso intencional de tipografía variable (opcional).

Repositorio en GitHub

  • Repositorio público.
  • Archivo README.md con descripción conceptual.
  • Captura del Hero incluida en el README.
  • Enlace al deploy (GitHub Pages).
  • Commits descriptivos.
  • Estructura clara de carpetas (css, img, assets).

Rúbrica de Evaluación

1. Hero (25 pts)

  • Jerarquía visual clara (5)
  • Uso correcto de layout (5)
  • Uso de clases e ids (5)
  • Adaptabilidad móvil (5)
  • Coherencia tipográfica (5)

2. Estructura HTML (15 pts)

  • Uso semántico correcto (5)
  • Jerarquía de títulos (5)
  • Accesibilidad básica (5)

3. Layout y Modelo de Caja (15 pts)

  • Uso adecuado de spacing (5)
  • Uso de flex o grid (5)
  • Ritmo y alineación consistentes (5)

4. Mobile First (10 pts)

  • Funciona correctamente en móvil (5)
  • Media queries bien aplicadas (5)

5. Sistema Visual (10 pts)

  • Escala tipográfica coherente (5)
  • Estados de interacción claros (5)

6. Multimedia y Optimización (10 pts)

  • Imágenes optimizadas (5)
  • Formatos modernos con fallback (5)

7. Accesibilidad (5 pts)

  • Contraste adecuado (3)
  • Estados focus visibles (2)

8. Repositorio y Documentación (10 pts)

  • README claro (5)
  • Commits descriptivos (5)
